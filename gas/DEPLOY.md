# GAS デプロイガイド (v2: デッキ単位ファイル方式)

## ⚠️ 旧バックアップデータについて

このバージョンから、バックアップ形式が「1セルに全データ」から「デッキ単位ファイル」に変わりました。
**旧形式のバックアップデータは引き継がれません。**
アップデート後、設定画面の「今すぐバックアップ」を押して新形式で保存し直してください。

---

## セットアップ手順

### 1. Google Drive にバックアップフォルダを作成

1. [Google Drive](https://drive.google.com/) を開く
2. 任意の場所に新規フォルダを作成（例: `RapidCycleBackup`）
3. フォルダを開き、URL からフォルダIDを取得
   - URL 例: `https://drive.google.com/drive/folders/1AbCdEfGhIjKlMnOpQrStUvWx`
   - フォルダID: `1AbCdEfGhIjKlMnOpQrStUvWx`（`folders/` の後ろの部分）

### 2. GAS プロジェクトを作成

**方法A: スプレッドシートにバインド（推奨）**
1. Google スプレッドシートを新規作成
2. **拡張機能 → Apps Script** を開く
3. `gas/Code.gs` の内容をエディタに貼り付ける

**方法B: スタンドアロン**
1. [script.google.com](https://script.google.com/) で新規プロジェクトを作成
2. `gas/Code.gs` の内容をエディタに貼り付ける

### 3. FOLDER_ID を差し替え

`Code.gs` の先頭にある定数を自分のフォルダIDに変更：

```javascript
var FOLDER_ID = "1AbCdEfGhIjKlMnOpQrStUvWx"; // ← 自分のIDに変更
```

### 4. デプロイ

1. エディタ上部「**デプロイ**」→「**新しいデプロイ**」
2. 設定:
   - **種類**: ウェブアプリ
   - **次のユーザーとして実行**: 自分
   - **アクセスできるユーザー**: 全員
3. 「**デプロイ**」をクリック → 承認画面が出たら許可する
4. 表示された **ウェブアプリのURL** をコピー

> **注意**: 旧バージョンの GAS がある場合、URL が変わる可能性があります。
> アプリ側でも URL を更新してください。

### 5. アプリに URL を設定

1. Rapid Cycle の **設定 → クラウドバックアップ (GAS)**
2. コピーした URL を貼り付け
3. **「今すぐバックアップ」** を押す
4. Drive フォルダに `meta.json` と `deck_*.json` が作成されれば成功

---

## コードを更新した場合の再デプロイ

`Code.gs` を変更したときは「新しいバージョン」としてデプロイし直す必要があります：

1. **デプロイ** → **デプロイを管理**
2. 編集（鉛筆アイコン）→ バージョン: **「新しいバージョン」** を選択
3. デプロイ

URL は変わりません（同じ URL で新コードが動きます）。

---

## Drive フォルダの構成

バックアップ後、フォルダには以下のファイルが作成されます：

```
RapidCycleBackup/
├── meta.json          — フォルダ情報
├── deck_<id1>.json    — 単語帳1（単語 + 学習記録）
├── deck_<id2>.json    — 単語帳2
└── ...
```

各ファイルは「今すぐバックアップ」のたびに上書きされます。
削除したデッキのファイルはゴミ箱に移動されます（P3以降で自動化）。
